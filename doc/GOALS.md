# DAC Web - 项目目标

## 概述

DAC Web 是一个基于浏览器的数据分析应用，通过 DAC（Data Action Context）框架，让用户能够在网页界面中进行数据处理和分析。

## 核心功能

### 1. 项目管理
- 创建新的分析项目
- 保存和加载项目配置
- 发布/共享项目配置
- 通过签名追踪项目历史

### 2. 会话管理
- 每个项目运行在独立的子进程会话中
- 会话通过 UUID 标识
- 会话在运行期间保持状态（上下文、数据、动作）
- 会话终止时可记录输出日志

### 3. 上下文管理
- 支持多种上下文类型（上下文用于分组数据和动作）
- 全局上下文（GCK）作为默认上下文
- 支持上下文切换
- 通过 YAML 配置各上下文

### 4. 数据管理
- 在上下文中存储和管理数据节点
- 数据类型由 DAC 框架定义
- 数据可被可视化（如 matplotlib 集成）

### 5. 动作管理
- 动作是在上下文中执行的分析操作
- 支持多种动作类型（由场景定义）
- 动作有生命周期状态：新建 → 已配置 → 已完成/失败
- 动作可以产生新的数据节点

### 6. 场景系统
- 场景定义可用的上下文类型和动作类型
- 用户可以在不同场景间切换
- 默认场景：`0.base.yaml`

### 7. 可视化
- 集成 matplotlib 进行数据可视化
- 在网页界面中显示交互式图表

## 架构

### 前端（dac-frontend）
- Svelte 5 + TypeScript 单页应用
- 基于路由的导航（新建项目、加载项目）
- YAML 编辑器用于配置
- 实时可视化画布

### 后端（backend）
- FastAPI 服务器
- PostgreSQL 存储会话元数据
- 为每个分析会话生成子进程
- 反向代理到会话子进程

### 存储结构
```
storage/
├── projects/      # 活动项目配置
├── projects_save/ # 已发布/共享的项目
└── logs/         # 会话输出日志
```

## 用户工作流程

1. 用户打开 `/projects/new` 开始新分析
2. 系统创建会话并返回会话 ID
3. 用户选择场景并创建上下文
4. 用户在上下文中添加和配置动作
5. 用户运行动作处理数据
6. 结果可可视化或保存
7. 项目可保存备用或发布

## 技术栈

- **前端**：Svelte 5、TypeScript、sv-router、axios
- **后端**：Python 3.12+、FastAPI、Pydantic、asyncpg
- **核心**：miz-dac 框架
- **数据库**：PostgreSQL
- **容器**：Docker/Podman
